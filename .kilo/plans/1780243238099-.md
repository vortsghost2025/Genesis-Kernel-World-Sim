# SkillUI Evaluation Plan for Genesis-Kernel-World-Sim

## Goal
Evaluate SkillUI as a UI phenotype extraction tool to establish a repeatable UI quality workflow that prevents ugly maps/dashboards/mobile pages.

## Constraints
- Do not change simulation logic
- Do not modify Mom's Sim
- Do not touch provider keys or NIM routing
- Do not deploy
- Do not install global tools unless necessary; prefer sandbox/tooling folder
- No production behavior changes
- No secrets captured
- No provider settings captured

### Phase 4: Artifact Collection
Save only useful extracted artifacts under: `docs/ui-phenotypes/`
- WORLD_SIM_DASHBOARD_SKILL.md
- WORLD_SIM_MAP_SKILL.md
- UI_TOKENS.md or tokens JSON
- COMPONENTS.md
- LAYOUT.md
- screenshots/

### Phase 5: Documentation Creation
1. Create: `docs/UI_QUALITY_CONTRACT.md` with requirements:
   - HTTP 200 is not UI success
   - DOM selectors existing is not UI success
   - Every UI task needs desktop screenshot and mobile screenshot
   - No overlapping text
   - No tiny unreadable map labels
   - No hardcoded future/locked places visible by default
   - No provider/debug/admin text in user-facing views
   - Default screen must answer: what is this, what matters, what do I do next?

2. Create: `docs/prompts/UI_REVIEW_GATE.md`
   - Copy-paste prompt for future agents to review UI before claiming completion

### Phase 6: Reporting
Report on:
- Whether SkillUI output is useful or noisy
- Which docs should be kept
- Whether it improved agent instructions
- Recommendation: adopt / reject / use only occasionally

## Acceptance Criteria
- No production behavior changes
- No secrets captured
- No provider settings captured
- Screenshots saved
- UI quality contract exists
- UI review gate prompt exists
- Clear recommendation provided
